关于本项目
----------

这是一个使用Sphinx生成的开源文档项目 [1]_，源代码托管在GitHub仓库上，现在所看到的网页托管在Read
the Docs。

本项目主要存放笔记，我会陆续将一些过去所做的笔记整理后上传到本项目，并不定期更新相关笔记。

创建本项目的初衷是为了存放笔记，关于电子笔记这件事多少还是有些经验了。

.. admonition:: OneNote
   :class: note
   
   首先接触并使用过的是\ ``OneNote``\ ，是我用过的很好的一款笔记软件，但是对公式支持不是很友好，并且同步效果也不算很理想。

.. admonition:: LaTeX
   :class: note

   而后转战到了\ ``LaTeX``\ ，这是一款排版软件，用来记笔记属实是有点大材小用了，完美的解决了公式问题，做出来的笔记排版美观、赏心悦目，后来我还用它排版过课程论文和项目申报书，效果也是绝好的。

.. admonition:: Markdown
   :class: note

   再后来接触到了\ ``Markdown``\ ，可以理解是极简版的\ ``LaTeX``\ 。同样对公式极度友好，操作简单易上手，大概十五分钟就能学会并熟练使用。记一些简单的笔记或者当做草稿也是极好的。

.. admonition:: reStructuredText
   :class: note

   现在又接触到了\ ``reStructuredText``\ ，我的理解是它介于\ ``Markdown``\ 和\ ``LaTeX``\ 之间，高于\ ``Markdown``\ 的可玩性，低于\ ``LaTeX``\ 的复杂性。本项目的文档就是由它写成。

上述工具除了\ ``OneNote``\ 和\ ``reStructuredText``\ 外，都不支持同步和随时查阅的需要。尝试过其他的解决办法，最终确定这应该是目前的最优解。

这是我创建的第二个博客，相当于人生中的第二套房了，尽管现实中我一套房都没有😞。

我想把关于生活的记录放在第一个博客，关于学习和工作的笔记放在这个博客。不做精确划分两者会有交集，同样欢迎你访问我的另一个博客\ `QLX <https://qiulinxin.github.io/>`__\ 🎉。

就目前而言，我会将侧重点首先放在这个博客上，未来将同时兼顾两个博客的发展。

感想
----

前面说了一大堆的废话，很高兴你能够读到这里🌹😄。

我以前特别喜欢玩一款游戏\ ``我的世界``\ ，游戏中有\ ``生存``\ 和\ ``创造``\ 两种模式，我更喜欢\ ``创造``\ 模式。

在里面建房子、修铁路和港口、种小麦和甘蔗、喂马、钓鱼，这就是“我的世界”。躲在雪山的小屋里看雪，骑着马在草原驰骋，在海底漫游看珊瑚和各种鱼，这就是我所向往和喜欢的生活。

游戏终归是游戏，如果有一天游戏关闭或者数据丢失，游戏中创造的世界也就随之消失了。

我决定在现实中继续创造我的世界，写代码、搭建属于自己的博客，更新博客，随心的写点东西，这就是我的另一个世界。
